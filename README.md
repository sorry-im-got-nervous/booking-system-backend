# Booking System Backend

## Описание проекта

**Booking System Backend** — серверная часть системы управления бронированием общих ресурсов организации.

Проект предоставляет REST API для регистрации и авторизации пользователей, управления ресурсами, создания и отмены бронирований, просмотра расписания ресурсов, а также получения пользовательских уведомлений. Система предотвращает конфликты бронирования, поддерживает разграничение прав доступа между пользователями и администраторами и обеспечивает сохранение истории операций без физического удаления данных.

Проект разработан в рамках производственной практики.

---

## Основные возможности

### Авторизация пользователей

* регистрация новых пользователей;
* вход в систему с использованием JWT;
* защита маршрутов с помощью JWT Authentication;
* разграничение ролей (**user** и **admin**).

### Управление ресурсами

* просмотр списка ресурсов;
* создание новых ресурсов (администратор);
* изменение информации о ресурсе (администратор);
* изменение статуса ресурса (`active` / `inactive`);
* назначение ответственного пользователя.

> Для сохранения истории бронирований ресурсы не удаляются из системы, а переводятся в статус **inactive**.

### Управление бронированиями

* создание бронирований;
* просмотр собственных бронирований;
* просмотр всех бронирований (администратор);
* отмена бронирования;
* сохранение истории отменённых бронирований.

### Уведомления

* автоматическое создание уведомления после успешного бронирования;
* автоматическое создание уведомления при отмене бронирования;
* просмотр списка уведомлений текущего пользователя;
* отметка уведомлений как прочитанных.

### Проверки бизнес-логики

При создании бронирования реализованы проверки:

* обязательных полей;
* существования ресурса;
* активности ресурса;
* корректности временного интервала;
* отсутствия пересечения с существующими бронированиями;
* запрет бронирования на прошедшее время.

### Просмотр расписания ресурсов

Система предоставляет возможность получить расписание выбранного ресурса на указанную дату. Возвращаются только активные бронирования, что позволяет определить занятость ресурса и использовать данные для отображения календаря бронирований.

## Установка и запуск

### 1. Клонировать репозиторий

```bash
git clone <repository-url>
```

### 2. Перейти в папку проекта

```bash
cd booking-system-backend
```

### 3. Установить зависимости

```bash
npm install
```

### 4. Создать базу данных MySQL

Например:

```sql
CREATE DATABASE booking_system;
```

### 5. Выполнить SQL-скрипты

Сначала выполнить:

```text
database/schema.sql
```

Затем:

```text
database/seed.sql
```

### 6. Создать файл .env

Скопировать файл:

```text
.env.example
```

в

```text
.env
```

и указать параметры подключения к базе данных.

Пример:

```env
PORT=3000

DB_HOST=localhost
DB_PORT=3306
DB_NAME=booking_system
DB_USER=root
DB_PASSWORD=

JWT_SECRET=your_secret_key
```

### 7. Запустить сервер

Режим разработки:

```bash
npm run dev
```

Рабочий режим:

```bash
npm start
```

После запуска сервер будет доступен по адресу:

```text
http://localhost:3000
```

## Тестирование API

Для тестирования проекта используется коллекция Postman.

Файлы находятся в каталоге:

```text
docs/postman/
```

Необходимо импортировать:

* Booking System.postman_collection.json
* Booking System.postman_environment.json

После выполнения запроса авторизации JWT-токен автоматически сохраняется в переменную окружения `current_token` и используется в последующих запросах.

## Автоматизированное тестирование

Для проверки корректности работы REST API разработаны автоматизированные интеграционные тесты с использованием Jest и Supertest.

Проверяются следующие сценарии:

* регистрация нового пользователя;
* предотвращение повторной регистрации;
* успешная авторизация;
* авторизация с неверными учетными данными;
* получение списка ресурсов;
* создание ресурса администратором;
* запрет создания ресурса пользователем без прав администратора;
* получение информации о ресурсе;
* создание бронирования;
* предотвращение повторного бронирования при пересечении времени.

Запуск тестов:

```bash
npm test

## Реализованные REST API

### Authentication

* POST `/api/auth/register`
* POST `/api/auth/login`

### Resources

* GET `/api/resources`
* GET `/api/resources/:id`
* GET `/api/resources/:id/schedule?data`
* POST `/api/resources`
* PUT `/api/resources/:id`

### Bookings

* POST `/api/bookings`
* GET `/api/bookings/my`
* GET `/api/bookings`
* PATCH `/api/bookings/:id/cancel`

### Notifications

* GET /api/notifications
* PATCH /api/notifications/:id/read

## Особенности реализации

В проекте реализованы следующие архитектурные решения:

* использование JWT для аутентификации пользователей;
* разграничение прав доступа с помощью middleware;
* хранение паролей в виде bcrypt-хэшей;
* предотвращение двойного бронирования ресурсов;
* сохранение истории бронирований посредством изменения статуса вместо удаления записей;
* отказ от физического удаления ресурсов для обеспечения целостности данных;
* система уведомлений о создании и отмене бронирований;
* получение расписания ресурса по выбранной дате;
* автоматизированное тестирование основных REST API с использованием Jest и Supertest.
